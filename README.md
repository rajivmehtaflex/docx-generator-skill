# DocX Generator Skill

A comprehensive browser-based Word document generator using Python-WASM (PyScript) that converts HTML/Markdown to professional .docx files with complex layouts.

## 🚀 Quick Start

### Option 1: Browser-Based (Recommended)
```bash
cd docx-generator-skill
python -m http.server 8000
# Open http://localhost:8000 in your browser
```

### Option 2: Python Package
```bash
pip install -r requirements.txt
python docx_generator.py
```

## 📋 Features

- **Browser-based**: Runs entirely in the browser using WASM
- **Multiple Formats**: Supports both Markdown and HTML input
- **Complex Layouts**: Headers, footers, page breaks, tables
- **Professional Output**: .docx files with full formatting
- **Codex Integration**: Ready for agentic AI workflows
- **Real-time Preview**: Live preview as you type
- **Custom Styling**: Document properties, headers, footers

## 🎯 Use Cases

1. **Document Generation**: Convert reports, articles, documentation to .docx
2. **Templates**: Create reusable document templates
3. **Batch Processing**: Generate multiple documents at once
4. **AI Integration**: Use with Codex for automated document creation
5. **Web Applications**: Embed in web apps for document export

## 📦 Building the .whl File

The `docx_generator` library is distributed as a Python wheel (`.whl`) for easy installation. The web UI in `index.html` loads this wheel via `micropip` at startup from `wheels/`. If you modify `docx_generator.py` on a remote machine, rebuild the wheel so the changes are reflected in the web UI.

### Build Command

```bash
# From the project root directory
python -m build --wheel

# Output: dist/docx_generator-1.0.0-py3-none-any.whl
```

### After Rebuilding

```bash
# 1. Copy the fresh wheel to the wheels/ directory for PyScript
cp dist/docx_generator-1.0.0-py3-none-any.whl wheels/

# 2. (Optional) Install locally to test
pip install dist/docx_generator-1.0.0-py3-none-any.whl

# 3. Verify it works
python -c "from docx_generator import DocxGenerator, generate_docx_from_markdown; print('OK')"

# 4. Commit to repo (if deploying via git)
git add wheels/ dist/ pyproject.toml src/
git commit -m "rebuild wheel after changes"
git push
```

### Prerequisites

```bash
pip install build wheel setuptools
```

### Wheel Contents

The wheel packages `docx_generator/__init__.py` as the `docx_generator` module with these dependencies declared:
- `python-docx >= 1.1.0`
- `markdown2 >= 2.4.0`
- `beautifulsoup4 >= 4.12.0`

### Why Two Locations? (`wheels/` vs `dist/`)

| Directory | Purpose | Git |
|-----------|---------|-----|
| `dist/`   | Build output, auto-generated | Ignored |
| `wheels/` | Deployment copy used by PyScript web UI | Tracked |

Always copy the rebuilt wheel from `dist/` to `wheels/` so the web UI picks up your changes.

## 💻 Usage

### Basic Web Usage
1. Open `index.html` in a browser
2. Enter your Markdown or HTML content
3. Configure options (title, author, headers, etc.)
4. Click "Generate DocX"
5. Download your document

### Python Library Usage
```python
from docx_generator import generate_docx_from_markdown

markdown_content = """
# My Document

This is **bold** and *italic* text.

## Features

- Feature 1
- Feature 2
- Feature 3
"""

options = {
    "title": "My Document",
    "author": "John Doe",
    "header": "Confidential",
    "footer": "Generated by AI",
    "page_number_format": "total",
    "add_toc": True,
    "enable_page_breaks": True
}

docx_base64 = generate_docx_from_markdown(markdown_content, options)
```

### Advanced Usage
```python
from docx_generator import DocxGenerator

generator = DocxGenerator()
generator.create_document(title="Custom Document", author="Your Name")

# Add cover page
generator.add_cover_page(
    title="Annual Report",
    subtitle="2024 Summary",
    author="Finance Department",
    date="January 2024"
)

# Add custom headers/footers
generator.add_header("Annual Report 2024", alignment="center")
generator.add_footer("Confidential", alignment="center")

# Process content
html_content = "<h1>Financial Summary</h1><p>Revenue increased by 25%...</p>"
generator.process_html_content(html_content)

# Get result
docx_base64 = generator.save_to_base64()
```

## 🔧 Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `title` | string | "Untitled Document" | Document title |
| `author` | string | "Unknown" | Document author |
| `header` | string | "" | Header text |
| `footer` | string | "" | Footer text |
| `page_number_format` | string | "none" | "none", "simple", "total" |

| `enable_page_breaks` | boolean | true | Convert hr to page breaks |

## 🐛 Troubleshooting

### WASM Loading Issues
- Check browser console for CORS errors
- Ensure you're running via a local server, not file://
- Try clearing browser cache

### Memory Errors
- Reduce document complexity
- Split large documents into smaller sections
- Close other browser tabs

### Formatting Issues
- Ensure HTML is well-formed
- Check that CSS is compatible with python-docx
- Test with simpler content first

## 🔒 Security Considerations
